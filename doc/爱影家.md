### 2.1 组件导航

Navigation是路由容器组件，一般作为首页的根容器，适用于模块内和跨模块的路由切换。Navigation组件主要包含导航页和子页面，导航页由标题栏、内容区和工具栏组成，可以通过hideNavBar属性控制导航页的显示和隐藏，导航页不存在页面栈中，导航页和子页，以及子页之间可以通过路由操作进行切换。Navtigation路由相关的操作都是基于页面栈NavPathStack提供的方法进行，每个Navigation都需要创建并传入一个NavPathStack对象，用于管理页面。主要涉及页面跳转、页面返回、页面替换、页面删除、参数获取、路由拦截等功能。

在爱影家应用程序的入口页面`IMovie/entry/src/main/ets/pages/Index.ets`中使用Navtigation组件作为首页的根容器，并为Navigation组件绑定路由栈`pageStack`。通过Navigation组件的`mode`属性设置导航栏的显示模式，默认为基于组件宽度自适应单栏和双栏`NavigationMode.Auto`，在当前应用中，设置为导航栏与内容区独立显示`NavigationMode.Stack`，相当于两个页面。通过`hideTitleBar`属性设置隐藏标题栏，`hideToolBar`属性设置隐藏工具栏，`hideBackButton`属性设置隐藏标题栏中的返回按钮。详细代码如下所示：

```typescript
@Entry
@Component
struct Index {

  // 路由栈
  pageStack: NavPathStack = new NavPathStack();

  build() {
    Navigation(this.pageStack) {
      // 具体页面实现
    }
    .width(IMConstants.PAGE_PERCENT)
    .height(IMConstants.PAGE_PERCENT)
    // 显示模式
    .mode(NavigationMode.Stack)
    // 隐藏标题栏
    .hideTitleBar(true)
    // 隐藏工具栏
    .hideToolBar(true)
    // 隐藏返回按钮
    .hideBackButton(true)
  }
}
```

NavDestination是Navigation子页面的根容器，可以设置独立的标题栏和菜单栏等属性，使用方法与Navigation相同。以搜索为例，若不使用NavDestination构建子页面，返回时会直接退出应用程序，而不是预期的返回上一页。同时可以通过`onReady`接口获取页面栈信息，也可以通过`queryNavigationInfo`方法获取页面栈信息。

```typescript
@Component
struct SearchMovieView {
  pageStack: NavPathStack = new NavPathStack();
  
  build() {
    NavDestination() {
      
    }
    .title("搜索影视")
    .onReady((context: NavDestinationContext) => {
      this.pageStack = context.pathStack;
    })
  }

  aboutToAppear(): void {
    const navigationInfo: NavigationInfo = this.queryNavigationInfo() as NavigationInfo;
    this.pageStack = navigationInfo.pathStack;
  }
}
```

### 2.2 选项卡Tabs

Tabs组件可以实现通过页签进行内容视图切换的容器组件，不支持自定义组件作为子组件，仅可包含子组件TabContent，可以在TabContent中使用自定义组件。Tabs页签默认位于页面顶部，将`barPosition`属性设置为`BarPosition.End`可以使页签位于页面底部，将`scrollable`属性设置为false，使页签不能通过滑动方式切换页面。

TabContent组件作为Tabs组件的内容视图，可以通过`tabBar`属性设置页签上显示内容，使用构造器CustomBuilder自定义页签显示内容。详细代码如下所示：

```typescript
@Component
export struct Home {

  @State currentIndex: number = 0;

  @Builder
  TabBuilder(tabBarItem: TabBarItem, targetIndex: number) {
    Column({ space: IMConstants.COLUMN_ROW_SPACE[2] }) {
      Image(tabBarItem.icon)
        .size({ width: $r('app.float.size24'), height: $r('app.float.size24') })
        .fillColor(this.currentIndex === targetIndex ? $r('app.color.tab_selected_color') : $r('app.color.tab_default_color'))
      Text(tabBarItem.text)
        .fontSize($r('app.float.font_size14'))
        .fontColor(this.currentIndex === targetIndex ? $r('app.color.tab_selected_color') : $r('app.color.tab_default_color'))
    }
    .width(IMConstants.PAGE_PERCENT)
    .justifyContent(FlexAlign.Center)
  }

  build() {
    Tabs({ index: this.currentIndex, barPosition: BarPosition.End }) {
      TabContent() {
        HomeView()
      }
      .tabBar(this.TabBuilder(TAB_BAR[0], 0))
      TabContent() {
        CollectView()
      }
      .tabBar(this.TabBuilder(TAB_BAR[1], 1))
    }
    .width(IMConstants.PAGE_PERCENT)
    .height(IMConstants.PAGE_PERCENT)
    .scrollable(false)
    .onTabBarClick((index: number) => {
      this.currentIndex = index;
    })
  }
}
```

### 2.3 远场通信服务

远场通信服务（Remote Communication Kit）是华为提供的HTTP发起请求的NAPI封装接口，开发者可以通过编写少量的代码便捷快速的向服务器发起数据请求，提供了fetch方法发送网络请求，get方法发送网络请求，post方法发送网络请求等。由于应用程序中会有多次发起数据请求的需求，因此，我们选择将远场通信服务进行二次封装，实现代码复用以及方便后续的维护扩展。封装需要遵循ArkTS语言规范和接口返回数据的处理。

#### 定义接口返回公共参数类ApiResult

```typescript
// 该类使用泛型T，方便返回不同的请求数据，某些接口无返回属性均定义为可选
export interface ApiResult<T> {
  code: number;
  message: string;
  data: T;
  start?: number;
  count?: number;
  total?: number;
  urls?: string[];
  tvurls?: string[];
  title?: string;
  valid?: boolean;
}
```

#### 封装远场通信服务

```typescript
class DataRequestUtil {

  private session: rcp.Session;

  constructor() {
    this.session = rcp.createSession();
  }

  // 返回具体的请求数据
  get<T>(url: string): Promise<T> {
    return new Promise((resolve, reject) => {
      this.session.get(url).then((response) => {
        const result = response.toJSON() as ApiResult<T>;
        resolve(result.data);
      }).catch((error: BusinessError) => {
        reject(error);
      });
    });
  }

  // 直接返回ApiResult
  getObj<T>(url: string): Promise<ApiResult<T>> {
    return new Promise((resolve, reject) => {
      this.session.get(url).then((response) => {
        const result = response.toJSON() as ApiResult<T>;
        resolve(result);
      }).catch((error: BusinessError) => {
        reject(error);
      });
    });
  }

  // 直接返回ApiReuslt
  post<T>(url: string, data: RequestParamsModel): Promise<ApiResult<T>> {
    return new Promise((resolve, reject) => {
      this.session.post(url, data).then((response) => {
        const result = response.toJSON() as ApiResult<T>;
        resolve(result);
      }).catch((error: BusinessError) => {
        reject(error);
      })
    })
  }

  // 直接返回请求数据
  postObj<T>(url: string, data: RequestParamsModel): Promise<T> {
    return new Promise((resolve, reject) => {
      this.session.post(url, data).then((response) => {
        const result = response.toJSON() as T;
        resolve(result);
      }).catch((error: BusinessError) => {
        reject(error);
      })
    })
  }
}

export const HttpService = new DataRequestUtil();
```

### 2.7 即将上映

即将上映是一个可以左右滑动的影视列表，可以使用可滑动容器组件Scroll，设置滚动方向为水平`scrollable(ScrollDirection.Horizontal)`，同时禁用滚动条`scrollBar(BarState.Off)`，内置弹性布局容器Flex，子组件布局方向为行布局Row和子组件间隔。通过ForEach循环渲染方式将即将上映数据呈现出来。详细代码如下所示：

```typescript
@Component
export struct UpComingView {
  pageStack: NavPathStack = new NavPathStack();
  private upcomingScroller: Scroller = new Scroller();

  @State upcomingList: MovieModel[] = [];

  build() {
    Column() {
      ComposeListItem({
        contentItem: ({
          iconStyle: IconType.SYSTEM_ICON,
          icon: $r('app.media.ic_upcoming'),
          primaryText: "即将上映"
        }),
        operateItem: ({
          icon: {
            value: $r('app.media.ic_more'),
            action: () => {
              this.pageStack.pushPathByName("moreUpComing", null);
            }
          }
        })
      })

      Column() {
        Scroll(this.upcomingScroller) {
          Flex({ direction: FlexDirection.Row, space: { main: { value: IMConstants.COLUMN_ROW_SPACE[3], unit: LengthUnit.VP } } }) {
            ForEach(this.upcomingList, (item: MovieModel) => {
              Column({ space: IMConstants.COLUMN_ROW_SPACE[3] }) {
                Image(item.cover)
                  .width($r('app.float.size128'))
                  .height($r('app.float.size200'))
                  .objectFit(ImageFit.Cover)
                  .borderRadius($r('app.float.size8'))
                Text(item.title)
                  .width($r('app.float.size128'))
                  .fontSize($r('app.float.font_size12'))
                  .fontWeight(FontWeight.Bold)
                  .maxLines(1)
                  .textOverflow({ overflow: TextOverflow.Ellipsis })
                  .textAlign(TextAlign.Center)
                if (item.rate) {
                  Row({ space: IMConstants.COLUMN_ROW_SPACE[4] }) {
                    Rating({ rating: (item.rate / 2), indicator: true })
                    Text(`（${item.rate}）`)
                      .fontSize($r('app.float.font_size10'))
                      .fontColor($r('app.color.today_box_office_highlight'))
                  }
                  .width($r('app.float.size128'))
                  .justifyContent(FlexAlign.Center)
                }
              }
            })
          }
        }
        .width(IMConstants.PAGE_PERCENT)
        .scrollBar(BarState.Off)
        .scrollable(ScrollDirection.Horizontal)
      }
      .width(IMConstants.PAGE_PERCENT)
      .padding($r('app.float.size8'))
    }
    .width(IMConstants.PAGE_PERCENT)
```

### 2.8 页面跳转

无论是点击标题栏的搜索按钮，还是点击轮播图，或者影视卡片中的影视封面，都涉及到页面的跳转，该应用中使用Navigation支持的系统路由表方式进行动态路由管理，在路由配置文件route_map.json中配置路由信息后，在触发路由跳转时，应用只需要通过NavPathStack提供的路由方法，传入需要路由的页面配置名称，系统会自动完成路由模块的动态加载、页面组件的构建，并完成路由跳转，从而实现开发层面的模块解耦。

#### 添加路由表配置

首先需要在主模块配置文件module.json5中添加路由表配置信息：

```typescript
{
  "module": {
    ...
    "routerMap": "$profile:route_map"
  }
}
```

#### 配置路由表

在主模块resources/base/profile中创建路由配置表route_map.json文件，并配置跳转页面名称name、跳转目标页面在包内的路径、跳转目标页面的入口函数名称，以及其他自定义字段。

```typescript
{
  "routerMap": [
    {
      "name": "searchMovie",
      "pageSourceFile": "src/main/ets/components/SearchMovieView.ets",
      "buildFunction": "SearchMovieViewBuilder",
      "data": {
        "description": "影视搜索"
      }
    },
    ...
  ]
```

在跳转目标页面中，比如搜索页面，配置入口Builder函数，函数名称必须与route_map.json配置文件中的buildFunction保持一致。

### 3.2 使用懒加载LazyForEach渲染搜索结果

前面，我们使用了循环渲染ForEach语句渲染数据，对于加载数据量较大的列表数据，建议使用懒加载LazyForEach。该方法需要我们先定义加载数据源，我们首先编写一个泛型的公共数据源类，该类需要实现IDataSource接口，并重写totalCount、getData等方法，代码如下所示：

```typescript
export class BasicDataSource<T> implements IDataSource {

  private listeners: DataChangeListener[] = [];
  private originDataArray: T[] = [];

  totalCount(): number {
    return this.originDataArray.length;
  }

  getData(index: number): T {
    return this.originDataArray[index];
  }

  registerDataChangeListener(listener: DataChangeListener): void {
    if (this.listeners.indexOf(listener) < 0) {
      this.listeners.push(listener);
    }
  }

  unregisterDataChangeListener(listener: DataChangeListener): void {
    const pos = this.listeners.indexOf(listener);
    if (pos >= 0) {
      this.listeners.slice(pos, 1);
    }
  }

  // 通知LazyForEach组件需要重新重载所有子组件
  notifyDataReload(): void {
    this.listeners.forEach(listener => {
      listener.onDataReloaded();
    })
  }

  // 通知LazyForEach组件需要在index对应索引处添加子组件
  notifyDataAdd(index: number): void {
    this.listeners.forEach(listener => {
      listener.onDataAdd(index);
    })
  }
}
```

接下来，我们实现一个用于构建影视数据源的类，该类中我们需要自定义一个在列表末尾添加数据并通知监听器的pushData方法和重载数据的reloadData方法，实现代码如下所示：

```typescript
export class MovieDataSource<T> extends BasicDataSource<T> {

  private dataArray: T[] = [];

  totalCount(): number {
    return this.dataArray.length;
  }

  getData(index: number): T {
    return this.dataArray[index];
  }

  // 在列表末尾添加数据并通知监听器
  pushData(data: T): void {
    this.dataArray.push(data);
    this.notifyDataAdd(this.dataArray.length - 1);
  }

  // 重载数据
  reloadData(): void {
    // 不会引起状态变化
    this.dataArray = [];
    // 必须通过DataChangeListener来更新
    this.notifyDataReload();
  }
}
```

最后在搜索页面，获取搜索结果时，调用pushData方法向数据源中添加数据，改变搜索关键字后，首先重置搜索列表内容，然后再调用接口获取搜索结果。

```typescript
// 获取搜索数据
```
